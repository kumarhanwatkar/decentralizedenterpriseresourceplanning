## 🟦 STEP 2: Setup MongoDB Atlas (YOU DO THIS - 15-20 min)

### 2.1 Create MongoDB Atlas Account

```
Go to: https://www.mongodb.com/cloud/atlas/register

Fill in:
  Email: your-email@gmail.com
  Password: Strong password (save it!)
  Checkbox: I have read and agree to...

Click: Create your Atlas account

✉️ Check your email and verify!
```

**Time**: ~2 minutes

### 2.2 Create Free Cluster

```
After login:
  1. Click [Build a Database] or [Create]
  2. Choose [M0 Shared] tier (FREE, 512 MB)
  3. Cloud Provider: AWS (default is fine)
  4. Region: Pick one close to your location
  5. Cluster name: erp-cluster
  6. Click [Create Deployment]

⏳ Wait 2-3 minutes. Cluster will be ready...
```

**Time**: ~3-4 minutes (mostly waiting)

### 2.3 Create Database User

```
After cluster is ready:
  1. Click [Databases]
  2. Click [Security] > [Database Access]
  3. Click [Add New Database User]
  
  Fill in:
  - Username: erpuser
  - Autogenerated password (COPY THIS!)
  - Built-in role: [readWriteAnyDatabase]
  
  4. Click [Add User]

⚠️ SAVE the password somewhere safe!
```

**Time**: ~2 minutes

### 2.4 Allow Network Access

```
In MongoDB Atlas:
  1. Click [Security] > [Network Access]
  2. Click [Add IP Address]
  3. Click [Allow Access from Anywhere]
     (Safe because credentials are protected)
  4. Click [Confirm]
```

**Time**: ~1 minute

### 2.5 Get Connection String

```
Click [Databases]
Click [Connect] on your cluster
Select [Drivers] > [Node.js] > v4.x or later

Copy the connection string like:
mongodb+srv://erpuser:<password>@erp-cluster.xxxxx.mongodb.net/?retryWrites=true&w=majority

Replace <password> with your actual password:
mongodb+srv://erpuser:MyPassword123@erp-cluster.xxxxx.mongodb.net/?retryWrites=true&w=majority

✅ SAVE THIS - You need it next!
```

**Time**: ~2 minutes

## Troubleshooting

### "Deployment failed" in Render

Check the Logs tab:
1. If error is about dependencies: `npm ci` might be failing
2. If error is about compilation: Check TypeScript errors
3. If error is about env vars: Make sure MONGODB_URI is set

**Fix**: 
```bash
# Locally, test build:
cd backend
npm ci
npm run build
```

### MongoDB connection error

```
MongoNetworkError: failed to connect to server
```

Check:
1. MONGODB_URI has correct password
2. IP address whitelisted in MongoDB Atlas
3. Connection string includes `/erp-db` database name

### Frontend can't call backend

DevTools → Network tab → Check:
1. Is VITE_API_BASE_URL correct?
2. Are API requests going to right URL?
3. Any CORS errors shown?

**Fix**: Check CORS settings in backend environment

### Everything looks good but data doesn't load

1. Refresh frontend page (Ctrl+Shift+Del to clear cache)
2. Check browser console for error messages
3. Check Render backend logs for errors
4. Verify database user credentials
