1. PROJECT OVERVIEW
================================================================================

WHAT IS THIS PROJECT?
---------------------
This is a Decentralized ERP (Enterprise Resource Planning) system designed for 
TechForge Industries that replaces traditional, centralized ERP systems with a 
transparent, blockchain-based solution. Instead of monthly or bi-monthly paychecks, 
employees receive STREAMING PAYROLL - real-time salary payments that flow 
continuously every hour.

KEY INNOVATION: The 85/15 Split-Wallet Model
An employee's earnings are automatically split:
- 15% → Immediately accessible in personal wallet (cold wallet)
- 85% → Locked in smart contract for 15 days (hot wallet)

During the 15-day lock period, a high-performance algorithmic trading bot uses 
the locked funds to generate passive yield in crypto markets. After 15 days, 
the employee receives their full salary PLUS all generated profits.

BENEFITS:
- Zero payroll delay: Employees get paid EVERY HOUR, not monthly
- Passive income: The 85% locked portion generates yield automatically
- Full transparency: Every transaction recorded on Binance Smart Chain (BSC)
- Employee empowerment: Immediate access to 15% + profitable lock-up for 85%
- Organization efficiency: Reduces wage-holding float costs

THE PROBLEM IT SOLVES:
Traditional centralized ERPs:
✗ Force employees to wait 15-30 days for paychecks
✗ Create idle capital pools (money held that could generate returns)
✗ Are opaque and siloed
✗ Require expensive integration and months of setup
✗ Lock organizations into vendor lock-in

This D-ERP Solution:
✓ Real-time payments every hour
✓ Automatic yield generation on held funds
✓ Complete blockchain transparency
✓ AI-driven dashboard generation (no complex ERP setup needed)
✓ Democratizes access to institutional-grade financial strategies

2. SYSTEM ARCHITECTURE
================================================================================

TECHNOLOGY STACK:
-----------------

FRONTEND:
- React 18.3.1: Modern UI framework for building interactive dashboards
- TypeScript 5.8.3: Type-safe JavaScript for reduced errors
- Vite 5.4.19: Lightning-fast build tool (alternative to Webpack)
- Tailwind CSS 3.4.17: Utility-first styling framework
- Shadcn/UI: Pre-built, customizable React components
- Framer Motion: Smooth animations and transitions
- Recharts 2.15.4: Professional charts and graphs
- Lucide Icons: Beautiful SVG icons for UI elements

BLOCKCHAIN & WEB3:
- Binance Smart Chain (BSC): Blockchain network for smart contracts
- ethers.js 6.16.0: JavaScript library to interact with Ethereum/BSC
- MetaMask: User wallet for signing transactions
- Smart Contracts: Solidity programs that handle payroll distribution

STATE MANAGEMENT:
- React Context API: Central store for all application data
- OrganizationContext: Holds all organization, employee, and transaction data
- AuthContext: Manages user authentication and roles
- Web3Context: Manages blockchain connection and wallet data
- ThemeContext: Manages dark/light mode

DATA FLOW ARCHITECTURE:
-----------------------

1. AUTHENTICATION LAYER:
   User connects MetaMask wallet
   ↓
   AuthContext validates wallet address
   ↓
   User is assigned role: Admin or Employee
   ↓
   Auto-redirect to appropriate dashboard

2. DATA LAYER:
   OrganizationContext (Central State Store)
   ├── Employees (5 employees with streaming payroll)
   ├── Resources (5 assets with utilization metrics)
   ├── Transactions (4 blockchain transactions)
   ├── Settings (Organization configuration)
   └── Stats (Real-time calculations)

3. REAL-TIME UPDATE LAYER:
   useEffect hook triggers every 1 second
   ↓
   Payroll stream increments by (hourlyRate / 3600)
   ↓
   All connected pages update automatically
   ↓
   No page refresh needed (React re-renders only changed components)

4. BLOCKCHAIN LAYER:
   Smart Contract on BSC
   ├── Receives funds from organization treasury
   ├── Splits 15% to cold wallet, 85% to hot wallet
   ├── Triggers algorithmic trading bot on 85%
   ├── Records every transaction immutably
   └── Returns profit after 15-day lock period

COMPONENT HIERARCHY:
--------------------

App (Root)
├── QueryClientProvider (React Query)
├── ThemeProvider (Dark/Light mode)
├── Web3Provider (Blockchain connection)
├── AuthProvider (User authentication)
├── OrganizationProvider (Central state store)
└── Routes
    ├── / (Landing Page)
    ├── /login (Login Page)
    ├── /admin (Admin Dashboard)
    │   ├── /admin/payroll (Payroll Management)
    │   ├── /admin/resources (Resource Tracking)
    │   ├── /admin/transactions (Blockchain Ledger)
    │   ├── /admin/ai-config (AI Dashboard Generation)
    │   └── /admin/settings (Organization Settings)
    ├── /employee (Employee Dashboard)
    │   ├── /employee/earnings (Earnings Analysis)
    │   ├── /employee/transactions (Personal Transactions)
    │   └── /employee/settings (Account Management)
    └── * (404 - Not Found)

ERROR 2: TRANSACTION FAILURES
---------------------------

Error Message: "Transaction failed: insufficient funds"
Problem: Not enough USDT/BUSD or BNB for gas fees
Solution:
1. Check your wallet balance in MetaMask
2. If USDT/BUSD too low:
   - Get more from exchange
   - Send to your wallet address
   - Wait 5-30 minutes for arrival
3. If BNB too low (for gas):
   - You need at least 0.01 BNB for one transaction
   - Buy more BNB and send to wallet
   - Re-attempt transaction after arrival

---

Error Message: "Transaction pending too long"
Problem: Transaction stuck for 30+ minutes without confirming
Solution:
1. Refresh browser page
2. Check BSCscan status:
   - Get your transaction hash
   - Go to https://bscscan.com
   - Paste hash in search
   - Check status (might show as confirmed)
   If confirmed, problem is just slow app update
3. If still pending:
   - Clear browser cache (Ctrl+Shift+Delete)
   - Restart browser
   - Try transaction again with slight increase in gas price

---

Error Message: "Transaction reverted: contract error"
Problem: Smart contract rejected the transaction
Solution:
1. Check transaction details in MetaMask:
   - Are amounts correct?
   - Is recipient correct?
   - Is contract address correct?
2. If details wrong:
   - Cancel transaction
   - Wait for full cancellation
   - Try again with correct details
3. If details correct but still failing:
   - Increase gas price to "Fast"
   - Retry transaction
   - If all else fails, contact support

DEPLOYMENT TO GITHUB PAGES:

GitHub Pages allows free hosting of static websites.

Prerequisites:

- Project pushed to GitHub repository

Deployment Steps:

   ```
   export default defineConfig({
     base: '/repository-name/',
     plugins: [react()],
     // ...
   })
   ```

2. Build project:
   npm run build

3. Commit and push to GitHub:
   git add .
   git commit -m "Build for deployment"
   git push origin main

   - Go to GitHub repository settings
   - Scroll to "GitHub Pages"
   - Select "Deploy from a branch"
   - Choose branch: "main" (or "master")
   - Choose folder: "/dist" (or "/docs" if you added dist to docs)
   - Click "Save"

5. Wait 1-3 minutes for deployment
   Your app is now live at: https://username.github.io/repository-name/

6. Verify deployment:
   - Visit the URL shown in GitHub Pages settings
   - Test all features work
   - Verify images, styles, data loads correctly

         - run: npm run build
         - run: npm run deploy

   - Deploys to hosting
